# TUI Design Specification

The `ixchel ui` terminal user interface provides an interactive dashboard for
browsing, searching, and managing the knowledge graph.

Note: this spec was drafted under the working name â€œIxchelâ€. The implemented tool
is **Ixchel** (binary `ixchel`, canonical dir `.ixchel/`). Treat `ixchel` â†’ `ixchel`
and `.ixchel/` â†’ `.ixchel/` when reading. The TUI is not implemented yet.

## Design Principles

1. **Keyboard-First**: Full functionality without mouse
2. **Vim-Like Navigation**: j/k, gg/G, /search
3. **Context-Aware**: Show relevant information based on selection
4. **Fast**: Sub-100ms response times
5. **Minimal Chrome**: Maximum content, minimal decoration

## Technology Stack

- **Framework**: [ratatui](https://github.com/ratatui-org/ratatui)
- **Event Loop**: crossterm backend
- **Async**: tokio for background operations
- **State Management**: Elm-inspired architecture

## Launch

```bash
ixchel ui                    # Full interface
ixchel ui --view browser     # Start in browser view
ixchel ui --view search      # Start in search view
ixchel ui --entity dec-42    # Jump to specific entity
```

## State Management

### Application State

```rust
pub struct App {
    // Current view
    view: View,
    view_stack: Vec<View>,  // For back navigation

    // Data
    entities: EntityCache,
    search_results: Option<SearchResults>,
    graph: Option<GraphData>,

    // UI state
    sidebar: SidebarState,
    browser: BrowserState,
    detail: DetailState,
    search: SearchState,
    graph_view: GraphViewState,
    editor: EditorState,

    // Background
    sync_status: SyncStatus,
    pending_operations: Vec<Operation>,
}

pub enum View {
    Browser,
    Detail(EntityId),
    Search,
    Graph(EntityId),
    Create(EntityType),
    Edit(EntityId),
    Dashboard,
    Help,
}

pub struct BrowserState {
    selected_type: Option<EntityType>,
    filters: Filters,
    sort: SortConfig,
    list_state: ListState,  // Scroll position
}

pub struct SearchState {
    query: String,
    cursor: usize,
    filters: Filters,
    results: Vec<SearchResult>,
    selected: usize,
}
```

### Event Loop

```rust
loop {
    // Render current state
    terminal.draw(|f| ui::render(f, &app))?;

    // Handle input with timeout (for background updates)
    if event::poll(Duration::from_millis(100))? {
        match event::read()? {
            Event::Key(key) => {
                let action = keybindings::handle(key, &app);
                app.dispatch(action);
            }
            Event::Resize(w, h) => {
                app.resize(w, h);
            }
            _ => {}
        }
    }

    // Process background updates
    while let Ok(update) = background_rx.try_recv() {
        app.apply_update(update);
    }
}
```
