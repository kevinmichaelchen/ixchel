# Ixchel Distribution & Publishing Report

**Date:** 2026-01-19
**Status:** Research Complete
**Tags:** distribution, publishing, packaging, ci

## Executive Summary

This report catalogs all viable methods to distribute ixchel's three Rust binaries (`ix-cli`, `ixcheld`, `ixchel-mcp`) to end users. Methods are organized by category, with recommendations based on audience reach, maintenance burden, and ecosystem fit.

---

## What Gets Published?

| Binary       | Crate Name | Description                         |
| ------------ | ---------- | ----------------------------------- |
| `ixchel`     | ix-cli     | Main CLI for knowledge management   |
| `ixcheld`    | ix-daemon  | Per-user daemon for sync/IPC        |
| `ixchel-mcp` | ix-mcp     | MCP server for AI agent integration |

**Build Artifacts:**

- Pre-compiled binaries (tarball/zip per platform)
- Shell/PowerShell installers
- Platform-specific packages (deb, rpm, pkg, msi)
- Container images (optional)

---

## Distribution Methods Catalog

### Tier 1: Rust Ecosystem (Primary Audience)

#### 1. crates.io + `cargo install`

**Where:** https://crates.io

```bash
cargo install ix-cli ix-daemon ix-mcp
```

| Aspect      | Details                                    |
| ----------- | ------------------------------------------ |
| Audience    | Rust developers                            |
| Maintenance | Minimal (`cargo publish`)                  |
| Pros        | Standard, discoverable, versioned          |
| Cons        | Compiles from source (slow), requires Rust |

#### 2. cargo-binstall

**Where:** GitHub Releases + crates.io metadata

```bash
cargo binstall ix-cli
```

| Aspect      | Details                                                        |
| ----------- | -------------------------------------------------------------- |
| Audience    | Rust developers with binstall                                  |
| Maintenance | Low (CI builds binaries)                                       |
| Pros        | Fast install, falls back to cargo install                      |
| Cons        | Requires cargo-binstall installed                              |
| Reference   | [cargo-binstall](https://github.com/cargo-bins/cargo-binstall) |

**Cargo.toml metadata:**

```toml
[package.metadata.binstall]
pkg-url = "{ repo }/releases/download/v{ version }/{ name }-{ version }-{ target }{ archive-suffix }"
pkg-fmt = "tgz"
```

#### 3. cargo-dist (Recommended for Rust)

**Where:** GitHub Releases + installers

```bash
# Shell installer (generated by cargo-dist)
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/kevinmichaelchen/ixchel/releases/download/v0.1.0/ix-cli-installer.sh | sh

# PowerShell installer
powershell -ExecutionPolicy ByPass -c "irm https://github.com/kevinmichaelchen/ixchel/releases/download/v0.1.0/ix-cli-installer.ps1 | iex"
```

| Aspect      | Details                                                               |
| ----------- | --------------------------------------------------------------------- |
| Audience    | All users (no Rust required)                                          |
| Maintenance | Low (generates CI + installers)                                       |
| Pros        | Cross-platform installers, SBOM, signing, Homebrew/npm/msi generation |
| Cons        | Rust-specific tooling                                                 |
| Reference   | [cargo-dist docs](https://axodotdev.github.io/cargo-dist/)            |

**Features:**

- Generates release.yml CI workflow
- Creates shell/PowerShell installers
- Supports Homebrew formula generation
- SBOM (CycloneDX) generation
- GitHub Artifact Attestations
- Native arm64 Linux runners

### Tier 2: System Package Managers

#### 4. Homebrew (macOS/Linux)

**Where:** homebrew-core or custom tap

```bash
# Official (requires acceptance)
brew install ixchel

# Custom tap (immediate)
brew install kevinmichaelchen/tap/ixchel
```

| Aspect      | Details                                                                                                             |
| ----------- | ------------------------------------------------------------------------------------------------------------------- |
| Audience    | macOS users, some Linux                                                                                             |
| Maintenance | Medium (formula updates per release)                                                                                |
| Pros        | Most familiar to macOS devs, auto-updates                                                                           |
| Cons        | Tap requires separate repo, bottles need CI                                                                         |
| Reference   | [Homebrew Tap Guide](https://casraf.dev/2025/01/distribute-open-source-tools-with-homebrew-taps-a-beginners-guide/) |

**Automation:** cargo-dist or GoReleaser can auto-generate formulae.

#### 5. Scoop (Windows)

**Where:** scoop bucket

```powershell
scoop bucket add ixchel https://github.com/kevinmichaelchen/scoop-ixchel
scoop install ixchel
```

| Aspect      | Details                                                         |
| ----------- | --------------------------------------------------------------- |
| Audience    | Windows developers                                              |
| Maintenance | Low (JSON manifest)                                             |
| Pros        | Native Windows package manager feel                             |
| Cons        | Smaller audience than Homebrew                                  |
| Reference   | [GoReleaser Scoop](https://goreleaser.com/customization/scoop/) |

#### 6. Winget (Windows)

**Where:** microsoft/winget-pkgs

```powershell
winget install kevinmichaelchen.ixchel
```

| Aspect      | Details                                                           |
| ----------- | ----------------------------------------------------------------- |
| Audience    | Windows 10/11 users                                               |
| Maintenance | Medium (PR to winget-pkgs)                                        |
| Pros        | Built into Windows, official feel                                 |
| Cons        | Requires PR approval for each release                             |
| Reference   | [GoReleaser Winget](https://goreleaser.com/customization/winget/) |

#### 7. AUR (Arch Linux)

**Where:** Arch User Repository

```bash
# Using yay
yay -S ixchel-bin

# Manual
git clone https://aur.archlinux.org/ixchel-bin.git
cd ixchel-bin && makepkg -si
```

| Aspect      | Details                                            |
| ----------- | -------------------------------------------------- |
| Audience    | Arch Linux users                                   |
| Maintenance | Low (PKGBUILD + .SRCINFO)                          |
| Pros        | Strong Arch community, cargo-aur automates         |
| Cons        | Only Arch-based distros                            |
| Reference   | [cargo-aur](https://github.com/fosskers/cargo-aur) |

#### 8. APT/DEB (Debian/Ubuntu)

**Where:** PPA or direct .deb download

```bash
# Direct download
curl -LO https://github.com/kevinmichaelchen/ixchel/releases/download/v0.1.0/ixchel_0.1.0_amd64.deb
sudo dpkg -i ixchel_0.1.0_amd64.deb

# PPA (requires Launchpad setup)
sudo add-apt-repository ppa:kevinmichaelchen/ixchel
sudo apt update && sudo apt install ixchel
```

| Aspect      | Details                                                 |
| ----------- | ------------------------------------------------------- |
| Audience    | Debian/Ubuntu users                                     |
| Maintenance | High (PPA) or Low (direct .deb)                         |
| Pros        | Native package management                               |
| Cons        | PPA setup complex, separate repo                        |
| Reference   | [nFPM](https://nfpm.goreleaser.com/) for deb generation |

#### 9. RPM (Fedora/RHEL)

**Where:** COPR or direct .rpm download

```bash
# Direct download
curl -LO https://github.com/kevinmichaelchen/ixchel/releases/download/v0.1.0/ixchel-0.1.0.x86_64.rpm
sudo rpm -i ixchel-0.1.0.x86_64.rpm
```

| Aspect      | Details                  |
| ----------- | ------------------------ |
| Audience    | Fedora/RHEL/CentOS users |
| Maintenance | Medium                   |
| Pros        | Enterprise Linux support |
| Cons        | Smaller desktop audience |

## Recommended Strategy

### Phase 1: Foundation (Immediate)

| Method          | Priority | Effort |
| --------------- | -------- | ------ |
| crates.io       | Must     | Low    |
| cargo-binstall  | Must     | Low    |
| cargo-dist      | Should   | Low    |
| GitHub Releases | Must     | Auto   |

**Why cargo-dist:** Single tool that generates:

- Cross-platform binaries
- Shell + PowerShell installers
- Homebrew formula (optional)
- npm package (optional)
- SBOM and attestations

### Phase 2: Expand Reach (Based on Demand)

| Method             | Priority | Trigger                  |
| ------------------ | -------- | ------------------------ |
| Homebrew tap       | High     | macOS user requests      |
| Nix flake          | Medium   | Nix user requests        |
| AUR                | Medium   | Arch user requests       |
| mise/aqua registry | Low      | Auto-supported via cargo |

### Phase 3: Broader Audience (Future)

| Method       | Priority | Trigger                         |
| ------------ | -------- | ------------------------------- |
| npm wrapper  | Low      | Web dev interest                |
| PyPI wrapper | Low      | Python dev interest             |
| winget       | Low      | Windows enterprise demand       |
| nixpkgs PR   | Low      | Established Nix community usage |

## CI/CD Tooling Comparison

| Tool            | Language           | Outputs                             | Complexity |
| --------------- | ------------------ | ----------------------------------- | ---------- |
| **cargo-dist**  | Rust               | Binaries, installers, Homebrew, npm | Low        |
| **release-plz** | Rust               | crates.io, changelog, tags          | Low        |
| **GoReleaser**  | Go (can wrap Rust) | Everything                          | Medium     |
| **nFPM**        | Any                | deb, rpm, apk, archlinux            | Low        |
| **cargo-aur**   | Rust               | AUR PKGBUILD                        | Low        |

**Recommendation:** Use **cargo-dist** for binaries + installers, **release-plz** for crates.io automation.
